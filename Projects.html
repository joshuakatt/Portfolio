Rubiks Cube Visualizer

Interactive 3D model of Rubiks cube using Python and Matplotlib

Github

As a child, I always found Rubik's Cubes intriguing. The seemingly infinite faces a cube could have, the directions in which it could be twisted, and of course, the ways in which one could solve the cube.

I developed this 3D visualization program using Python and Matplotlib (which I happened to use in one of my courses). You can move the cube around in 3D, rotate faces of the cube and undo your steps. Eventually, I plan to make this a software to teach people how to solve a cube. I plan to implement this by guiding the user in the right steps and warning the user when wrong steps are being made.

You can find the code to this Project here: Github

I also made a version of it with a Pyraminx (tetrahedron-shaped puzzle with four triangular faces), let me know if you're interested in seeing this!

CrazyRydes Site

Serverless web application using AWS Lambda, Amazon API Gateway, Amazon DynamoDB, Amazon Cognito, and AWS Amplify Console.

Visit Website

CrazyRydes Site is a serverless web application built using various AWS services including AWS Lambda, Amazon API Gateway, AWS Amplify, Amazon DynamoDB, and Amazon Cognito. It provides a scalable and secure solution for building web applications without the need to manage servers.

This project demonstrates the power and flexibility of serverless architecture in modern web development. The backend is built using AWS Lambda and Amazon API Gateway, enabling automatic scaling and reduced operational overhead. User authentication is implemented using Amazon Cognito, ensuring a secure and scalable way to manage user sign-up, sign-in, and access control. Data storage and retrieval are handled by Amazon DynamoDB, a fast and flexible NoSQL database service.

AWS Amplify is used for frontend development, providing tools and services to streamline the development process, including hosting and deployment.

Ride Booking App

A powerful React Native application for seamless ride booking, inspired by leading ride-hailing services and leveraging Google's Directions API, Locations API, and other essential Google APIs.

Visit Website

The "Ride Booking App" is a powerful React Native application that offers a seamless ride booking experience, inspired by leading ride-hailing services. The app is designed to streamline transportation and provide users with an efficient and user-friendly interface for booking rides in real-time.

The app leverages Google's Directions API to provide optimized routes, ensuring efficient and time-saving trips for riders. The Google Locations API accurately tracks users' locations, enabling seamless real-time navigation. The interactive map interface allows users to visualize available drivers, nearby locations, and ongoing trips. Real-time driver tracking enhances transparency, allowing users to monitor their drivers throughout the journey.

Other key features include access to ride history for past trips and relevant details, fare estimation functionality for trip cost estimation before booking, and a highly customizable interface that allows for branding and additional custom features.

Project Vendo

Social Media Managing Platform (In early development)

Learn More

My grandfather, mother, and I used to conduct music lessons for children in our neighborhood, and we always found managing advertising and social media extremely complex and time-consuming.

I thought that there should be a coordinated, easy, and time-saving solution to manage digital engagements for small businesses, but for all the platforms we tried, we could never find a cheap solution that catered to a small business like ours.

I'm currently in the initial stages of development where I'm figuring out how the Meta/Facebook Graph API and my backend will connect to allow users to log in and connect social media accounts.

I plan to allow users to connect their social media accounts (Facebook, Instagram) and schedule when their posts go out. More importantly, I wish to provide the user with insights on which type of posts are doing best, and how to reach a wider audience using analytics. I plan to implement this by guiding the user in the right steps and warning the user when wrong steps are being made.
